# Manual Testing Project – GIGA OBS Automobile Application

## 📌 Project Overview

This project focuses on **manual testing of a multi-module automobile service application** based on a detailed Software Requirements Specification (SRS). The application supports customers, service providers, and administrators, enabling service booking, product purchasing, and account management.

---

## 📄 Test Basis

* Software Requirements Specification (SRS) document
* Functional requirements including:

  * User registration and login (OTP-based)
  * Service booking and product ordering
  * Payment processing
  * Role-based access (Customer, Service Provider, Admin)
  * Feedback and notification systems

---

## 🧪 Testing Approach

* Requirement-based testing
* Functional testing
* Regression testing
* End-to-end workflow validation

---

## 📊 Test Artifacts

* ✅ **400+ Test Cases** covering multiple modules and scenarios
* ✅ **Requirement Traceability Matrix (RTM)** for requirement coverage
* ✅ Test scenarios for:

  * Authentication (OTP login & registration)
  * Form validation
  * Navigation and UI behavior
  * Payment workflows
  * Service booking and order management

---

## 🧩 Modules Covered

* **Customer Module**

  * Registration, login, profile management
  * Service booking and product ordering
  * Payment and feedback

* **Service Provider Module**

  * Registration and verification
  * Service management and dashboard
  * Purchase tracking

* **Admin Module**

  * User and service provider management
  * Order and service management
  * Reporting and system configuration

---

## 🛠 Tools Used

* Manual Testing
* Microsoft Excel (Test Case Documentation)

---

## 🎯 Key Outcomes

* Ensured comprehensive test coverage using RTM
* Validated complex workflows across multiple user roles
* Identified gaps between requirements and expected behavior
* Strengthened understanding of real-world QA processes and documentation
